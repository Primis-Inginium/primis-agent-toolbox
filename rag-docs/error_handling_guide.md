# 🚨 Primis-Inginium Error Handling & Logging

AI agents must follow these standards to ensure consistent troubleshooting across all apps.

## 1. Structured Errors
- Always use the `AppError` class from `@core/errors` (if available).
- Never "swallow" errors. Always log them or re-throw.

## 2. Logging Levels
- **INFO**: General application flow.
- **WARN**: Unexpected but non-breaking behavior.
- **ERROR**: Breaking issues (API failures, crashes).

## 3. Contextual Metadata
When logging, always include:
- `featureName`: The feature name (e.g., `auth`).
- `agentContext`: If the code was generated by an agent, include the agent version.

## 4. API Failures
- All API calls must include a fallback or a user-friendly error message.
- Use `@core/api` utilities which handle retries automatically.
